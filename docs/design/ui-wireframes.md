# UI 页面结构说明

> 状态：Active
> Owner：RAG 团队
> Last Updated：2026-06-29
> Scope：Knowledge Island Web MVP 前端页面布局与核心交互
> Related：docs/requirements/functional-modules.md, docs/design/api-spec.md

## 1. 页面清单

| 页面 | 目标用户 | 主要目标 | 优先级 |
|------|----------|----------|--------|
| 工作台（主页）| 所有用户 | RAG 问答、多轮对话、Agent 工具、检索复盘 | P0 |
| 资料库 | 所有用户 | 导入文档、管理集合、查看健康状态、批次历史 | P0 |
| 评估 | 学习者 | 生成题目、逐题作答、查看掌握情况、答题记录和待复测列表 | P1 |
| 设置 | 技术用户 | 配置 LLM、模型 Profile、Prompt 预设、数据备份 | P1 |

## 2. 布局表达约定

```text
┌─────────────────────────────┐
│  [区域名称]  表示 UI 功能区  │  ← 方括号为区域标签，非代码
└─────────────────────────────┘
```

Vue 主体验采用左侧工作区导航，页面通过侧栏标签切换视图（SPA，不刷新页面）。B-142 工作台目标结构参考 AnythingLLM 的工作区模式，但只吸收信息架构：左侧项目/会话，中间对话，右侧上下文；不复刻多用户、Agent 写操作、Agent Flow 或模型路由。

页面支持系统深色偏好；用户也可以通过侧栏主题按钮在浅色 / 深色之间手动切换，手动选择会保存到浏览器 `localStorage`。

## 3. 工作台（主页）

**页面目标**：支持多轮对话式 RAG 问答，查看来源和工具结果。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ [左侧工作区]              │ [中间对话区]                    │ [右侧上下文]   │
│ 知识岛                    │ ┌──────────────────────────────┐ │ 来源           │
│ 当前项目 ▼                │ │ [用户消息]  消息内容          │ │ 检索           │
│ 健康摘要 / 最近活动       │ │ [AI 回答]  流式 Markdown      │ │ 工具           │
│                            │ │ [可观察性条] hits/top_k/model │ │ ───────────── │
│ 工作台                    │ │ [工具建议横条] search_sources │ │ 命中文档       │
│ 资料库                    │ │ [反馈按钮] 有用 / 来源不准    │ │ 分数 / 片段    │
│ 评估                      │ └──────────────────────────────┘ │ 检索参数       │
│ 设置                      │ [输入框] 在此输入问题... [发送] │ 工具运行历史   │
│                            │ [取消当前回答]                  │                │
│ 会话                      │                                  │                │
│ + 新建会话                │                                  │                │
│ ● 架构说明                │                                  │                │
│   接口排查                │                                  │                │
│   导入复盘                │                                  │                │
└──────────────────────────────────────────────────────────────────────────────┘
```

**关键交互**：
- 切换会话后重新加载该会话聊天记录（`GET /api/chat/messages?session_id=`）
- 发送问题后按钮进入 loading 状态，禁用重复提交，并显示“取消”按钮关闭当前 EventSource 流
- 来源不足时显示建议工具横条，用户点击才触发工具运行（不自动执行）
- 工具运行成功后，可一键引用 `tool_run_id` 到下一轮问答
- 回答通过 `/api/answer/stream` 的 SSE `token` 事件逐步渲染，完成后用 `done` 事件刷新来源、观察性和聊天记录
- 检索调试、检索复盘和 Agent 只读工具默认进入右侧上下文栏，不占用中间对话主流程

**空状态**：暂无会话时显示"点击 + 新建会话开始问答"引导文案

**错误状态**：API 不可用时显示可恢复提示，说明下一步操作（检查服务 / 刷新）

## 4. 资料库

**页面目标**：让用户管理知识来源，查看知识库健康状态。

```text
┌────────────────────────────────────────────────────────────────┐
│  [顶部导航栏]                                                   │
├────────────────────────────────────────────────────────────────┤
│  [知识库辅助管理]                                               │
│   [项目状态] 文档/Chunk/向量/评估题数/最近活动                  │
│   [检索健康] 可检索 / 待补齐向量 / 知识库为空                   │
│   [摄入进度] 最近导入批次状态、创建/更新/删除/跳过计数          │
│   [文件列表] 最近文件快照；无文档时显示“知识库为空”             │
│   [评估题库] 题目数与题型分布                                   │
│   [最近结果] 结果数与掌握状态分布                               │
├────────────────────────────────────────────────────────────────┤
│  [导入操作区]                                                   │
│   [同步目录]  [上传文件]  [文本笔记]  [URL 摘录]  [剪贴板]     │
│  [导入预检]  预计新增 12 / 跳过 3（格式不支持 2 / 过大 1）     │
│  [最近导入批次]  目录同步 · 完成 · 新增12/跳过3 · 05-25 14:31 │
├────────────────────────────────────────────────────────────────┤
│  [集合筛选]  全部(42) | 未分组(15) | 后端项目(18) | 笔记(9)   │
│             [+ 新建集合]                                        │
│  [文档列表]                                                     │
│   📄 README.md          note:        2026-05-25   [删除]       │
│   📄 api-spec.md        local:docs/  2026-05-24   [删除]       │
│   📄 架构笔记            note:        2026-05-23   [删除]       │
└────────────────────────────────────────────────────────────────┘
```

**关键交互**：
- 打开资料库或切换项目后，顶部管理概览读取 `GET /api/projects/summary` 和 `GET /api/assessment/library`
- 导入、同步目录、删除文档、开始评估或提交评估回答后刷新管理概览
- 点击"同步目录"后按钮置为 loading，返回导入统计（新增/更新/跳过/失败）
- 导入前可点击"预检"查看估算结果，不写数据库
- 集合筛选即时过滤文档列表，无需刷新
- 删除文档前二次确认

**空状态**：未选择项目时显示"请选择项目空间后查看知识库管理概览"；无文档时显示"知识库为空"；无导入批次时显示"暂无导入批次历史"；无评估题目时显示"暂无评估题目"。

## 5. 评估页

**页面目标**：出题、作答、查看掌握等级和错题记录。

```text
┌────────────────────────────────────────────────────────────────┐
│  [顶部导航栏]                                                   │
├────────────────────────────────────────────────────────────────┤
│  [评估入口]                                                     │
│   当前项目：知识岛 Web MVP     [开始新一轮评估]                 │
│   上次评估：2026-05-24  已掌握 3 / 基本理解 2 / 需补充 1       │
├────────────────────────────────────────────────────────────────┤
│  [题目区]                                                       │
│   第 2 / 5 题                                                   │
│   ─────────────────────────────────────────────────────────── │
│   请描述 RAG 混合检索的流程，以及关键词检索和向量检索如何合并。 │
│   ─────────────────────────────────────────────────────────── │
│   [答案输入框（多行）]                                          │
│                                                                 │
│   [跳过]                              [提交回答]               │
├────────────────────────────────────────────────────────────────┤
│  [评估结果]（提交后显示）                                       │
│   🟡 基本理解    参考来源：docs/design/search.md § 3           │
│   补充说明：回答提到了关键词检索但未说明得分合并策略           │
```

**当前实现**：Web 前端已支持一轮多题评估的进度展示、逐题提交、单题反馈、答题记录和待复测列表；跨刷新持久化的错题复测入口仍待后续任务细化。

**空状态**：项目无文档时提示"请先在资料库导入文件再开始评估"

## 6. 设置页

**页面目标**：配置 LLM 连接、模型 Profile 和 Prompt 预设。

```text
┌────────────────────────────────────────────────────────────────┐
│  [顶部导航栏]                                                   │
├────────────────────────────────────────────────────────────────┤
│  [模型设置]                                                     │
│   API 地址  [https://api.deepseek.com/v1        ]              │
│   模型名称  [deepseek-chat                      ]              │
│   API Key   [已配置 env:DEEPSEEK_API_KEY        ]              │
│   [测试连接]                    [保存设置]                      │
│   ✅ 连接成功 · deepseek-chat · api                            │
├────────────────────────────────────────────────────────────────┤
│  [模型 Profile]                                                 │
│   ● DeepSeek（默认）  deepseek-chat  api.deepseek.com   [编辑] [删除] │
│     OpenAI GPT-4o    gpt-4o         api.openai.com      [编辑] [删除] │
│   [+ 新增 Profile]                                             │
├────────────────────────────────────────────────────────────────┤
│  [Prompt 预设]（当前项目：知识岛 Web MVP）                      │
│   ● 项目问答（默认）   简洁技术回答风格                [编辑] [删除] │
│     代码解释           逐步解释代码意图               [编辑] [删除] │
│     学习复盘           苏格拉底式追问                 [编辑] [删除] │
│   [+ 新增预设]                                                 │
├────────────────────────────────────────────────────────────────┤
│  [数据备份]                                                     │
│   [导出当前项目备份]           [从备份文件恢复]                 │
└────────────────────────────────────────────────────────────────┘
```

**关键交互**：
- 测试连接不改变配置，仅验证可达性
- API Key 回显只显示来源引用（`env:DEEPSEEK_API_KEY`），不显示明文
- Profile 设为默认后，下次问答优先使用该 Profile

## 7. 空状态与错误状态约定

| 场景 | 展示文案 | 下一步引导 |
|------|----------|------------|
| 未创建项目 | "尚无项目空间" | 跳转设置页创建 |
| 项目无文档 | "知识库为空" | 跳转资料库导入 |
| 检索无命中 | "未找到相关来源" | 建议换关键词或补充资料 |
| API 不可用 | "本地服务暂时不可访问" | 提示刷新或检查 `app.py` |
| LLM 未配置 | "未配置模型，使用本地片段回答" | 跳转设置页配置 API |
