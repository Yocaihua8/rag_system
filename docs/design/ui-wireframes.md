# UI 页面结构说明

> 状态：Active
> Owner：RAG 团队
> Last Updated：2026-07-02
> Scope：Knowledge Island Web MVP 前端页面布局与核心交互
> Related：docs/requirements/functional-modules.md, docs/design/api-spec.md

## 1. 页面清单

| 入口 | 目标用户 | 主要目标 | 优先级 |
|------|----------|----------|--------|
| 聊（默认首页） | 所有用户 | 围绕当前工作区资料聊天、整理、查看回答依据 | P0 |
| 库（弹窗） | 所有用户 | 加入资料、查看加入结果、选择已有资料 | P0 |
| 设（全屏页） | 所有用户 | 管理回答方式、在线连接、资料位置、备份恢复和外观 | P1 |

独立一级 `评估/练习` 入口已取消。练习与小测作为聊天输入框左下角 `+` 菜单中的工具进入；旧 `AssessmentView` 组件保留为后续聊天工具复用，不再作为主导航页面。

## 2. 布局表达约定

```text
┌─────────────────────────────┐
│  [区域名称]  表示 UI 功能区  │  ← 方括号为区域标签，非代码
└─────────────────────────────┘
```

Vue 主体验采用左侧工作区和线程侧栏，页面通过 `聊 / 库 / 设` 主入口切换主流程。B-156 后，`库` 不切换普通页面，而是打开资料管理弹窗；`设` 是全屏覆盖页。B-142 工作台目标结构参考 AnythingLLM 的工作区模式，但只吸收信息架构：左侧工作区/线程，中间对话，右侧依据；不复刻多用户、Agent 写操作、Agent Flow 或模型路由。

页面主色采用黑白灰中性方案；错误和警告状态保留必要提示色。深色模式入口位于 `设 -> 外观`。

## 3. 聊（主页）

**页面目标**：支持多轮对话式 RAG 问答，查看来源和工具结果。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ [左侧侧栏]                │ [中间聊天区]                    │ [依据抽屉]     │
│ Knowledge Island          │ ┌──────────────────────────────┐ │ 默认收起       │
│ 聊 / 库 / 设              │ │ [用户消息]  消息内容          │ │ 回答来源       │
│                            │ │ [AI 回答]  流式 Markdown      │ │ 来源质量       │
│ 工作区                    │ │ [反馈按钮] 有用 / 来源不准    │ │ 检索复盘       │
│ 当前工作区                │ └──────────────────────────────┘ │ 工具结果       │
│ [加入资料]                │ [输入框]                        │ 对比回答       │
│ 线程                      │ [+] [快捷工具] [回答引擎] [发送]│                │
│ [搜索线程]                │                                  │                │
│ + 新建                    │                                  │                │
│ ● 第二阶段前端设计        │                                  │                │
│   资料导入问题            │                                  │                │
└──────────────────────────────────────────────────────────────────────────────┘
```

**关键交互**：
- 切换线程后重新加载该线程聊天记录（`GET /api/chat/messages?session_id=`）
- 点击 `库` 或聊天输入框 `+ -> 加入资料` 打开资料管理弹窗，不离开聊天主流程
- 发送问题后按钮进入 loading 状态，禁用重复提交，并显示“取消”按钮关闭当前 EventSource 流
- 来源不足时显示建议工具横条，用户点击才触发工具运行（不自动执行）
- 工具运行成功后，可一键引用 `tool_run_id` 到下一轮问答
- 回答通过 `/api/answer/stream` 的 SSE `token` 事件逐步渲染，完成后用 `done` 事件刷新来源、观察性和聊天记录
- 来源、检索复盘、工具结果和对比回答默认进入右侧依据抽屉，不占用中间对话主流程
- 移动端默认隐藏侧栏，只保留顶部菜单按钮；点击后以抽屉显示侧栏

**空状态**：暂无会话时显示"点击 + 新建会话开始问答"引导文案

**错误状态**：API 不可用时显示可恢复提示，说明下一步操作（检查服务 / 刷新）

## 4. 库弹窗

**页面目标**：让用户先加入资料，再从已有资料中选择需要用于工作区聊天的内容。第一版不把资料库作为一级普通页面，也不承担问答主流程。

```text
┌──────────────────────────────────────────────────────────────────┐
│ [库弹窗] 管理资料                                      [关闭]     │
│ [加入库] [选择资料]                                               │
├──────────────────────────────────────────────────────────────────┤
│ 加入库                                                           │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐ │
│ │ 文件     │ │ 文件夹   │ │ 笔记     │ │ 网页摘录 │ │ 更多来源│ │
│ └──────────┘ └──────────┘ └──────────┘ └──────────┘ └────────┘ │
│ [本次结果] 默认折叠；导入完成或失败后自动展开                    │
├──────────────────────────────────────────────────────────────────┤
│ 选择资料                                                         │
│ 左侧主侧栏切换为“选择工作区”                                      │
│ 右侧显示资料夹、资料列表、加载、错误和空状态                      │
└────────────────────────────────────────────────────────────────┘
```

**关键交互**：
- `加入库` 首屏只展示文件、文件夹、笔记、网页摘录和更多来源
- GitHub、Notion、Obsidian 等高级来源收进“更多来源”；后端未完整支持时只显示说明或复用现有导入事件，不伪造成功
- `ImportResultList` 默认折叠；有导入状态或错误时自动展开
- 点击 `选择资料` 后，主左侧侧栏切换为工作区列表；选择目标工作区不切换当前聊天工作区
- 当前后端仍以 `project_id` 为资料边界，第一版不显示“已连接到多个工作区”这类状态

**空状态**：无资料时提示先用“加入库”上传文件、文件夹、笔记或网页。

## 5. 练习与小测

**页面目标**：能力保留，但不作为一级页面。第一版入口位于聊天输入框 `+ -> 练习与小测`，后续可复用旧评估组件作为聊天工具弹窗或抽屉。

```text
┌────────────────────────────────────────────────────────────────┐
│  [聊天工具] 练习与小测                                [关闭]    │
├────────────────────────────────────────────────────────────────┤
│   当前工作区：知识岛 Web MVP       [开始练习]                   │
│   上次结果：已掌握 3 / 基本理解 2 / 需补充 1                   │
├────────────────────────────────────────────────────────────────┤
│  [题目区]                                                       │
│   第 2 / 5 题                                                   │
│   ─────────────────────────────────────────────────────────── │
│   请描述 RAG 混合检索的流程，以及关键词检索和向量检索如何合并。 │
│   ─────────────────────────────────────────────────────────── │
│   [答案输入框（多行）]                                          │
│                                                                 │
│   [跳过]                              [提交回答]               │
├────────────────────────────────────────────────────────────────┤
│  [练习结果]（提交后显示）                                       │
│   🟡 基本理解    参考来源：docs/design/search.md § 3           │
│   补充说明：回答提到了关键词检索但未说明得分合并策略           │
│   [继续追问]              [下一题 →]                           │
├────────────────────────────────────────────────────────────────┤
│  [错题记录]  ▼ 展开                                             │
│   • 向量化降级策略（暂未掌握）— 下轮优先复测                   │
└────────────────────────────────────────────────────────────────┘
```

**当前实现**：Web 前端已支持一轮多题评估的进度展示、逐题提交、单题反馈、答题记录和待复测列表；跨刷新持久化的错题复测入口仍待后续任务细化。

**空状态**：项目无文档时提示"请先打开库加入资料，再开始练习"

## 6. 设置页

**页面目标**：全屏管理回答方式、资料保存、备份恢复和外观；技术字段默认收进详情。

```text
┌────────────────────────────────────────────────────────────────┐
│  [顶部导航栏]                                                   │
├────────────────────────────────────────────────────────────────┤
│  [返回] 设置                                                    │
│  [左侧菜单] 回答 / 资料 / 外观                                  │
├────────────────────────────────────────────────────────────────┤
│  [回答] 本机回答 / 本机快速 / 在线回答                          │
│        [连接详情] 服务地址、模型名、Key 引用、保存和测试          │
├────────────────────────────────────────────────────────────────┤
│  [资料] 资料保存位置 / 备份 / 恢复                              │
├────────────────────────────────────────────────────────────────┤
│  [外观] 深色模式 / 回答方式 / 依据显示                          │
└────────────────────────────────────────────────────────────────┘
```

**关键交互**：
- 测试连接不改变配置，仅验证可达性
- API Key 回显只显示来源引用（`env:DEEPSEEK_API_KEY`），不显示明文
- 每次只展示左侧菜单选中的一个设置页面，不做长滚动设置页
- 当前后端未提供完整资料保存位置迁移、备份恢复桌面能力时，按钮必须显示清楚的不可用或后续支持状态，不伪装成功

## 7. 空状态与错误状态约定

| 场景 | 展示文案 | 下一步引导 |
|------|----------|------------|
| 未创建项目 | "尚无项目空间" | 跳转设置页创建 |
| 项目无文档 | "知识库为空" | 打开库弹窗加入资料 |
| 检索无命中 | "未找到相关来源" | 建议换关键词或补充资料 |
| API 不可用 | "本地服务暂时不可访问" | 提示刷新或检查 `app.py` |
| 在线回答未配置 | "未配置在线回答，使用本地片段回答" | 打开设置页配置连接 |
