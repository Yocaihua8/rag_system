# MVP 范围冻结

> 状态：Active
> Owner：RAG 团队
> Last Updated：2026-05-16
> Scope：首批可交付范围
> Related：`requirements/functional-modules.md`

## 1. 纳入 MVP（当前可演示）

| 编号 | 功能 | 验收标准 |
|------|------|----------|
| MVP-01 | 项目空间导入与增量摄入 | 支持扫描、分块、重建与删除清理 |
| MVP-02 | Markdown 标准化与安全显示 | 生成三类内容，HTML 渲染不执行脚本 |
| MVP-03 | 检索 + 问答链路 | 可见答案与来源，支持上下文构建 |
| MVP-04 | 文档模型落地 | `Project/Document/Chunk/Tag/Source` 可持久化 |
| MVP-05 | 用户文案迁移 | UI 显示“项目空间”，兼容旧 Workspace 语义 |

## 2. 排除范围（当前版本不承诺）

| 编号 | 功能 | 原因 |
|------|------|------|
| OUT-01 | 完整 UI 知识掌握闭环 | 模型与用例完成，但页面与流程未全量接入 |
| OUT-02 | 全自动 OCR 与扫描件文档 | 需额外依赖链路与成本评估 |
| OUT-03 | 外部多用户协同 | 当前定位为个人知识工作流 |

## 3. 变更规则

- MVP 范围变化应先评估对 `design/` 与 `tests` 的影响。
- 修改范围时同步新增/调整 `BACKLOG.md` 条目并标注里程碑。
