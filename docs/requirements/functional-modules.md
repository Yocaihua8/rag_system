| 知识导入 | 将本地知识转为可检索资产 | 目录同步、文件上传、GitHub 仓库、笔记、URL 摘录、批次历史 | P0 |

| 掌握度评估 | 系统评估用户对知识的掌握程度 | 自动出题、回答评估、答题记录、待复测列表 | P1 |

## 2. 项目管理

### 2.1 模块目标

为用户的不同知识领域（代码项目、学习主题、工作笔记）提供隔离的工作空间。

### 2.2 核心功能

1. 创建项目空间，绑定本地目录或浏览器上传模式（`browser-upload:<name>`）
2. 重命名和删除项目（删除前二次确认）
3. 项目切换与最近项目恢复（页面刷新后自动恢复上次项目）
4. 保存项目级检索默认参数（top_k / min_score / 关键词开关 / 向量开关）
5. 项目健康概览（文档数、chunk 数、向量数、聊天数、工具运行数、复盘数、最近活动）

### 2.3 输入

- 项目名称、绑定目录路径（或 `browser-upload:<name>` 虚拟路径）

### 2.4 输出

- 项目列表、项目配置、项目健康摘要（`GET /api/projects/summary`）

### 2.5 边界说明

- 项目间数据相互隔离，不支持跨项目文档共享
- 不支持多用户权限管理（单机单用户）；B-118 研究结论为当前不实现团队空间，未来需先拆分身份、租户、RBAC、审计和迁移设计

## 3. 知识导入

### 3.1 模块目标

将本地文件、笔记和 URL 摘录以可检索格式入库。

### 3.2 核心功能

1. 本地目录同步（增量更新，自动忽略 `.git` / `node_modules` / `.venv` 等）
2. 浏览器文件夹导入（通过 `webkitdirectory` 在浏览器内选择并上传）
3. 单文件 / 多文件上传（PDF / DOCX / Markdown / TXT 及代码文件）
4. 文本笔记手动录入（`note:` 虚拟来源，同标题再次导入只更新）
5. URL 摘录（保存 URL + 标题 + 手动粘贴正文，`url:` 虚拟来源）
6. GitHub 仓库整体导入（通过本机 `git clone --depth 1` 拉取 GitHub 仓库并创建项目空间）
7. 剪贴板文本快速导入（复用笔记导入通道）
8. 导入预检（只读估算可导入数量和跳过原因，不写数据库）
9. 导入批次历史（记录每次导入结果、跳过明细和读取失败明细）

### 3.3 输入

- 本地目录路径 / 文件二进制内容（base64）/ 文本内容 / URL / GitHub 仓库 clone URL

### 3.4 输出

- 文档记录、文本 chunk、向量索引（实际 embedding 或本地 hash fallback）
- 导入统计（新增 / 更新 / 未变 / 删除 / 跳过 / 失败数）
- 导入批次记录（`import_batches` / `import_batch_items`）

### 3.5 边界说明

- 不支持网页自动爬取，URL 来源只保存手动粘贴的正文
- PDF 解析依赖可选的 `pymupdf`，未安装时明确跳过并说明原因
- 不支持 OCR（扫描件 PDF 无法提取文本）
- 单文件 > 1 MB 时跳过，防止阻塞导入流程
- 目录同步和浏览器文件夹导入不会删除或覆盖 `note:` / `url:` 虚拟来源
- GitHub 仓库导入只接受 GitHub clone URL，不接入 GitHub API，不保存 token、用户名、密码或 SSH Key；私有仓库访问依赖本机 git 已配置权限

## 4. 文档管理

### 4.1 模块目标

让用户查看、筛选和组织已导入的知识文档。

### 4.2 核心功能

1. 文档列表展示（来源路径、类型、更新时间）
2. 文档集合创建、编辑、删除
3. 文档加入 / 移出集合
4. 按集合过滤文档列表（全部 / 未分组 / 指定集合）
5. 删除文档

### 4.3 输入

- 文档 ID、集合名称、过滤条件（project_id + collection_id）

### 4.4 输出

- 文档列表、集合列表、关联关系

### 4.5 边界说明

- 集合只影响列表过滤，不改变检索范围（检索仍跨当前项目全库）
- 删除集合只清理关联关系，不删除文档本身
- 应用层保证集合与文档必须属于同一项目，跨项目加入集合必须拒绝

## 5. 检索与问答

### 5.1 模块目标

通过 RAG 将用户自然语言问题与知识库内容对接，给出有来源支撑的回答。

### 5.2 核心功能

1. 关键词检索（基于内置 BM25；英文 / 数字按 regex token，中文连续文本追加 bigram）
2. 向量检索（OpenAI-compatible embedding API 或本地 hash fallback）
3. 混合检索结果合并排序，取 top_k chunk
4. 多轮对话（每次回答携带最近 3 轮历史上下文）
5. 多会话管理（新建 / 切换 / 重命名 / 删除会话）
6. 本地 fallback 回答（无 LLM 时聚合 top chunk 片段返回）
7. 检索调试（临时调整 top_k / min_score / 开关，查看命中预览）
8. 回答质量反馈（有用 / 无用 / 来源不准 / 需更多上下文）
9. 流式输出（`/api/answer/stream` 通过 SSE / EventSource 逐步渲染回答）

### 5.3 输入

- 问题文本、会话 ID、项目 ID、可选 `tool_run_id`

### 5.4 输出

- 回答文本、流式 token、来源列表、可观察性元数据（hits / top_k / model / latency）

### 5.5 边界说明

- 非流式 `/api/answer` 仍保留为 JSON 兼容接口；前端默认使用 `/api/answer/stream`
- 来源引用精度依赖 chunk 质量，不支持精确页码定位
- 不支持对话分支或历史消息编辑重发（B-128，P2）

## 6. 掌握度评估

### 6.1 模块目标

帮助用户通过主动回答检验和提升对知识库内容的掌握程度。

### 6.2 核心功能

1. 基于项目文档自动生成评估题（概念理解 / 流程说明 / 代码定位）
2. 用户提交回答，AI 对照来源评估掌握等级
3. 等级输出：已掌握 / 基本理解 / 需要补充 / 暂未掌握
4. 错题记录与优先复测
5. 追问支持（题目级继续追问）
6. 能力差距报告

### 6.3 输入

- 项目 ID、用户回答文本

### 6.4 输出

- 题目列表、评估等级、参考来源、差距报告

### 6.5 边界说明

- Web MVP 已支持从已导入文档规则化生成概念理解、流程说明、代码定位三类评估题；前端可按题逐步作答、查看进度、单题反馈、答题记录和待复测题目；提交回答时会对照服务端保存的参考要点评估为已掌握 / 基本理解 / 需要补充 / 暂未掌握，并保存评估题、用户回答和评估结果
- 不支持自定义题型
- 待复测列表当前为本轮前端会话内展示，跨刷新持久化的错题复测入口仍待后续任务细化

## 7. Agent 工具

### 7.1 模块目标

在问答过程中提供只读的知识库辅助操作，增强回答质量。

### 7.2 核心功能

1. `project_overview` — 返回当前项目统计摘要（文档数 / chunk 数 / 向量数等）
2. `search_sources` — 对指定查询词执行检索，返回 top 命中片段
3. 工具运行历史查看（工具名 / 状态 / 参数 / 错误，审计记录）
4. 工具结果一键引用到下一轮问答（通过 `tool_run_id` 注入上下文）

### 7.3 边界说明

- 仅开放只读工具，硬编码白名单，不支持文件写入或 shell 命令
- 工具不自动执行，由用户手动触发
- 跨项目的 `tool_run_id` 会被拒绝注入

## 8. 模型与 Prompt 配置

### 8.1 模块目标

让用户灵活管理 LLM 连接配置和回答风格预设。

### 8.2 核心功能

1. 模型 Profile 多配置（保存多个 provider / API Base / 模型名组合）
2. 设置全局默认 Profile 用于问答
3. Profile 连接测试（不改变配置，不覆盖环境变量）
4. Prompt 预设（项目级回答风格模板：系统提示 + 回答格式）
5. 设置项目默认预设，在真实 LLM 回答时注入

### 8.3 边界说明

- Profile 只保存 API Key 引用（`env:RAG_LLM_API_KEY` / `saved:RAG_LLM_API_KEY` 等），不保存明文
- 不支持原生 Claude / Gemini SDK（仅支持 OpenAI-compatible 格式）
- 预设注入在固定来源约束之后，不改变检索参数

## 9. 可观察性

### 9.1 模块目标

让用户理解检索和回答的质量，支持持续改进知识库。

### 9.2 核心功能

1. 检索调试面板（命中 chunk / 分数 / 来源质量 / 上下文预览）
2. 检索复盘（保存诊断快照：查询词 + 参数 + 命中来源 + 人工备注）
3. 回答可观察性（每次回答展示：top_k / 命中数 / 模型模式 / 耗时）
4. 来源质量指示（来源充分 / 偏少 / 无来源）
5. 项目健康仪表盘（文档数 / chunk 数 / 向量数 / 活动统计）

### 9.3 边界说明

- 来源质量指示符不等于事实准确性评分
- 检索调试参数不持久化（调试后丢弃，不影响项目默认值）

## 10. 数据备份

### 10.1 模块目标

让用户保护和迁移知识库数据。

### 10.2 核心功能

1. 备份导出（只读返回：项目元数据 + 文档正文 + chunk/vector 快照 + 聊天记录 + 配置摘要）
2. 备份恢复（从同版本备份恢复为新的 `browser-upload:` 项目空间）

### 10.3 边界说明

- 导出和恢复文档正文、chunk / vector；恢复后无需重新导入即可检索问答
- 不导出或恢复 API Key，配置只保留 provider、API Base、模型名和 Key 是否已配置的摘要
- 跨版本备份恢复不作兼容保证
