# 2026-05-18 发布收口说明（B-53）

> 目标：确认知识掌握评估 8~10 阶段交付可发布状态
> Owner：RAG 团队
> 适用范围：桌面端知识岛（本地优先）

## 变更摘要

- 评估结果页支持“继续追问 / 错题复测”操作，形成题目级闭环。
- 错题尝试支持内存态记录，下一轮评估支持优先复测错题。
- 对关键失败路径增加可读异常提示，避免静默失败。
- 补齐回归测试，覆盖追问重答、错题清理、边界状态。

## 收口范围（本次可交付）

- **功能边界**
  - 评估链路：`开始评估` → `逐题回答` → `评估反馈` → `能力差距报告` → `继续追问` / `错题复测`。
  - 当前不承诺：
    - 自动难度自适应；
    - 跨会话个性化学习计划生成；
    - 分布式多用户协作与共享。
- **非承诺边界**
  - 不替代学习系统课程推荐；
  - 不承诺 OCR、拍照、远程云端知识库托管；
  - 非技术用户场景仍缺少安装包与正式启动引导文案。

### 手工体验验证（建议）

1. 导入有内容项目后，执行评估并在有至少 1 个薄弱点时观察“继续追问”可用；
2. 点击“错题复测”，确认有错题时优先出题；
3. 错误场景手工触发（如：未配置知识点时）是否给出清晰错误提示；
4. 在报告页未有追问问题时，追问按钮应保持禁用状态。

## 异常提示覆盖

- 会话未就绪：`请先重新开始本次评估`
- 题目不存在：`题目不存在，请重新开始评估`
- 追问题目缺失：`未找到可复测题目，请重新开始评估后选择题目`
- 无法生成题目：`未生成题目，请检查项目知识点是否已提取`
