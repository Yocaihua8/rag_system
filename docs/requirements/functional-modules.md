## 1. 模块清单

| 模块 | 当前状态 | 说明 |
|------|----------|------|
| 项目空间（Project） | 部分实现 | 新模型与存储落地，UI 文案已迁移，兼容层保留 `Workspace` |
| 文档导入与扫描 | 已实现 | 支持增量扫描、删除检测、checksum 去重、文件类型扩展 |
| Markdown 标准化 | 已实现 | 输出 `normalized_markdown / plain_text / rendered_html` |
| 检索问答 | 已实现 | 向量/关键词检索，支持上下文拼装与来源回显 |
| 生成能力（简历/JD/面试） | 兼容保留 | 在主线仍存在，但优先级低于项目知识线 |

| 掌握度与知识点 | 已建模+用例可用 | 数据模型与用例完成，UI 集成未完成 |
| 轻量知识图谱 | 已实现核心 | 节点/关系存储与查询落地，未完成 UI/闭环接入 |
| 非技术用户发布体验 | 待实现 | 体验清单见 `release/NON_TECH_RELEASE_CHECKLIST.md` |

## 2. 模块边界

- `src/application/*` 负责流程编排与事务边界。
- `src/adapters/*` 负责持久化、向量和 LLM 接入。
- `src/desktop/*` 负责 UI 交互，不直接承载规则决策。

## 3. 不纳入当前说明

- 模块拆解不在此处放置后续路线；后续需求改为 `BACKLOG.md` 项目条目。
- 任何未在上述模块状态表出现的功能默认为“研发中或规划中”。
