# Codex 式工作区会话与资料导入设计

> 状态：In Review
> Owner：RAG 团队
> Last Updated：2026-07-03
> Scope：第二阶段前端方向设计；定义下一版目标体验，不代表已实现行为。
> Related：docs/requirements/functional-modules.md, docs/design/api-spec.md, docs/design/chat-sessions-design.md, docs/design/import-batches-design.md, docs/BACKLOG.md

## 1. 目的

本文件定义 Knowledge Island 下一版前端方向：以 Codex 式工作区会话作为首页主线，以 AnythingLLM 式资料导入作为资料入口参考。

设计目标是让普通用户只理解一条路径：

```text
选工作区 -> 添加资料 -> 开会话 -> 提问 -> 看依据
```

本设计不复用迁移期前端页面结构，不把资料管理、检索调试、模型配置、评估结果和导入审计铺在首页。首页只服务当前会话。

## 2. 设计结论

下一版前端采用三类核心区域：

| 区域 | 用户理解 | 主要职责 |
|------|----------|----------|
| 工作区侧栏 | 我现在在哪个资料空间里工作 | 工作区切换、会话管理、添加资料入口 |
| 会话主区 | 我正在和资料聊天 | 展示当前会话、输入问题、接收回答 |
| 依据抽屉 | 这个回答参考了什么 | 展示来源、片段、来源不足提示和只读工具结果 |

导航只保留必要入口：

| 入口 | 定位 | 是否在首页展开 |
|------|------|----------------|
| 聊 | 默认首页，围绕当前工作区会话提问 | 是 |
| 资料 | 添加和浏览当前工作区资料 | 否，弹窗或独立页 |
| 设置 | 回答引擎、资料保存、备份和外观 | 否，全屏页 |

首页禁止做成管理后台。除当前会话所需信息外，其余能力一律收起。

当前可交互 HTML 预览：

- `docs/previews/codex-anythingllm-workspace-preview/index.html`

## 3. 命名规则

普通界面使用用户能直接理解的词：

| 内部或技术词 | 普通界面文案 | 说明 |
|--------------|--------------|------|
| project / 项目空间 | 工作区 | 用户理解为一组资料和会话 |
| session / 线程 | 会话 | 用户理解为一次聊天主题 |
| import / ingest | 添加资料 | 不使用“摄入” |
| model / LLM | 回答引擎 | 具体模型名放设置详情 |
| source / retrieval hit | 参考资料 | 回答依据的普通说法 |
| assessment | 练习一下 | 从聊天工具进入 |
| agent tool | 工具 | 只读能力，不暴露 Agent 概念 |

内部 API、数据库和代码可以继续使用 `project_id`、`session_id` 等现有命名。UI 文案变化不要求同步重命名底层字段。

术语边界：

- `docs/style-guide.md` 当前仍规定对外文案使用“项目空间”。
- 本文件处于 `In Review`，提出的是下一版普通界面文案调整：用“工作区”表达同一用户概念。
- 正式落地前，必须同步更新术语规范、前端文案和相关功能文档；在此之前，已实现行为文档仍以“项目空间”为准。

以下词不应出现在普通首页：

- RAG
- Agent
- Prompt
- Profile
- Chunk
- 向量
- top_k
- 检索调试
- Embedding
- Reranker

如确需展示，必须放入高级详情、设置页或调试入口。

## 4. 首页设计

### 4.1 首页只保留的信息

首页只展示当前会话必须的信息：

- 当前工作区
- 会话列表
- 当前会话内容
- 输入框
- 添加资料入口
- 回答引擎入口
- 依据入口

首页不展示：

- 项目健康统计
- 导入批次历史
- 文件管理列表
- 检索调试
- 工具运行历史
- 模型详细参数
- Prompt 预设
- 评估结果
- 备份恢复
- 数据库或向量状态

### 4.2 左侧工作区侧栏

左侧侧栏参考 Codex 的工作方式：

```text
Knowledge Island
[当前工作区 v]                 [添加资料]

会话
[搜索会话]
[+ 新建会话]

- 默认会话
- 接口设计讨论
- 资料导入问题
- 前端重设计

[资料] [设置]
```

行为规则：

- 点击当前工作区打开工作区切换面板。
- 新建会话必须归属当前工作区。
- 切换工作区后，会话、资料、回答来源和工具记录都切换到该工作区。
- 会话文案统一使用“会话”，不在普通界面使用“线程”。
- 默认会话在前端显示为“默认会话”；请求现有 API 时继续不传 `session_id`，兼容历史消息。

第一版不设计会话置顶、归档、收藏、跨工作区移动和智能自动命名。这些能力需要后续接口支持。

### 4.3 中间会话主区

会话主区只承载聊天：

```text
[会话标题]                                      [查看依据]

用户：这个项目的导入流程应该怎么设计？

回答：可以分为选择工作区、选择资料来源、预检、导入、查看结果五步...
      [参考资料 3]

┌────────────────────────────────────────────┐
│ 输入问题...                                │
│ [+] [工具]                       [本机 v] [发送] │
└────────────────────────────────────────────┘
```

输入框左侧：

- `+`：添加资料、上传临时文件、练习一下、整理要点。
- 工具：快捷指令、自定义工具、后续 MCP 入口。

输入框右侧：

- 回答引擎：本机、快速、在线。
- 思考程度：轻、标准、深入。
- 发送。

如果后端暂不支持单次思考程度，前端只能把它作为未生效的本地选择或设置入口，禁止表现成已经影响本轮回答。

### 4.4 右侧依据抽屉

依据抽屉默认收起，只在用户点击回答来源、顶部依据按钮或来源不足提示时打开。

抽屉内容按优先级展示：

1. 本次回答参考了哪些资料。
2. 每条来源的片段预览。
3. 来源不足时的补充建议。
4. 用户手动运行的只读工具结果。
5. 高级详情。

高级详情可以包含命中数、模型模式、耗时等信息，但默认不展示分数、chunk、向量、top_k 等技术字段。

## 5. 资料导入设计

资料导入参考 AnythingLLM 的“来源卡片 + 分步导入”体验，但不做连接器市场。

入口统一叫“添加资料”。

### 5.1 标准流程

所有资料入口遵循五步：

| 步骤 | 页面动作 | 说明 |
|------|----------|------|
| 1 | 选择目标工作区 | 没有工作区时先创建；不能无工作区问答 |
| 2 | 选择资料来源 | 用来源卡片分组展示 |
| 3 | 填写内容或预检 | 能预检的调用后端预检，不能预检的不伪装 |
| 4 | 开始导入 | 按钮进入等待状态，显示“正在处理资料” |
| 5 | 查看结果 | 展示摘要、刷新资料列表，提供“开始提问” |

后端当前没有通用导入任务进度流，因此导入时禁止显示百分比进度或后台队列。可以显示等待状态和当前动作说明。

### 5.2 来源分组

资料来源按普通用户理解分组：

| 分组 | 来源 | 当前边界 |
|------|------|----------|
| 本机资料 | 绑定目录同步 | 只对项目绑定的后端可见目录有效 |
| 本机资料 | 选择本机文件夹上传 | 浏览器授权后上传内容，不代表后续持续同步 |
| 本机资料 | 选择文件上传 | 支持一个或多个文件 |
| 本机资料 | Obsidian | 使用现有导入能力时展示；不可伪装账号连接 |
| 手写资料 | 文本笔记 | 写入当前工作区资料 |
| 手写资料 | 剪贴板文本 | 复用文本笔记能力 |
| 手写资料 | URL 摘录 | 保存 URL、标题和手动粘贴正文，不自动抓网页 |
| 网络资料 | 单网页抓取 | 只支持用户手动触发的单个公开网页，先预览再确认 |
| 网络资料 | GitHub 仓库 | 只接收 GitHub clone URL，通过本机 git 拉取；不接 GitHub API，不保存 token |
| 迁移资料 | Notion ZIP | 作为离线导入，不是 Notion 账号连接 |

未实现或能力不完整的来源不得显示为“已连接”“同步中”或“导入成功”。

### 5.3 导入结果

导入结果使用普通语言：

| 后端含义 | 普通文案 |
|----------|----------|
| created/imported | 已加入 |
| updated | 已更新 |
| unchanged | 已存在 |
| skipped | 已跳过 |
| error | 失败 |
| deleted | 已移除记录 |

展示结构：

1. 顶部摘要：已加入、已更新、已跳过、失败。
2. 明细分区：成功入库、普通跳过、读取失败。
3. 批次历史：默认折叠，只在资料管理中查看。

跳过不等于失败。格式不支持、超过大小限制、PDF 缺少解析器、扫描 PDF 无文本等应作为跳过或受控失败说明。

第一版禁止提供：

- 回滚批次。
- 重试批次。
- 删除批次。
- 精确导入进度。

### 5.4 已导入资料浏览

资料浏览属于资料区域，不进入首页主区。

资料区域支持：

- 当前工作区资料列表。
- 路径搜索。
- 集合过滤。
- 未分组过滤。
- 点击资料预览正文。
- 删除资料记录。
- 查看最近导入批次。

边界：

- 集合只影响资料列表过滤，不改变问答范围。
- 当前问答范围是当前工作区全库。
- 选择单个资料只能做预览或把文件名插入问题提示，不能表现成“本轮只问这个文件”。

## 6. 设置设计

设置使用全屏页面，不挤压聊天首页。

左侧为设置菜单，右侧一次只显示一个设置页：

| 设置页 | 内容 |
|--------|------|
| 回答引擎 | 本机、快速、在线回答选择 |
| 本机模型 | Ollama 状态、推荐模型、拉取入口 |
| 在线 API | OpenAI-compatible 地址、模型名、连接测试 |
| 访问密钥 | API Key 来源和状态，不回显明文 |
| 资料保存 | 当前保存位置、说明和后续迁移入口 |
| 备份恢复 | 导出和恢复当前工作区资料 |
| 外观 | 深色模式、界面密度 |
| 高级 | 检索参数、Prompt、Profile、调试入口 |

技术字段默认收起，只有用户进入对应设置页后才展示。

## 7. 练习设计

练习不作为一级入口。

入口位于聊天输入框 `+`：

- 练习一下。
- 检查掌握情况。
- 根据当前资料出题。

交互表现为会话中的工具：

1. 用户点击“练习一下”。
2. 系统基于当前工作区资料生成题目。
3. 用户直接在会话中回答。
4. 系统反馈掌握情况。

普通文案使用：

- 已掌握。
- 基本理解。
- 还需要补充。
- 暂时没掌握。

不在普通界面使用“掌握度评估模块”。

## 8. 后端和接口边界

### 8.1 可直接复用的能力

第一版可复用现有 API：

- 工作区列表、创建、改名、删除。
- 会话列表、新建、改名、删除。
- 消息列表、流式问答、回答反馈。
- 文件夹导入、文件上传、笔记、URL 摘录、网页抓取、GitHub 仓库、Obsidian、Notion ZIP。
- 文档列表、文档预览、文档删除、集合过滤。
- 导入批次列表和详情。
- 只读工具运行和工具结果回填。
- 模型设置、模型 Profile、Prompt 预设。
- 评估题库、开始评估、提交答案。
- 备份导出和恢复。

### 8.2 第一版禁止伪装的能力

以下能力需要后端新任务，前端第一版不能伪装：

- 跨工作区共享资料。
- 跨工作区问答。
- 只选择几个文件作为本轮回答范围。
- 集合作为检索范围。
- 通用导入实时进度。
- 会话置顶、归档、收藏。
- 自动智能会话命名。
- 批次回滚或重试。
- GitHub 账号连接或持续同步。
- MCP 工具市场。
- 多用户团队空间。

## 9. 响应式和桌面壳

桌面端优先：

- 左侧工作区侧栏固定。
- 用户点击收起后，左侧侧栏必须完全隐藏，不保留品牌窄栏或空白占位；顶部菜单按钮用于重新打开。
- 中间会话主区自适应。
- 右侧依据抽屉默认收起。
- 设置页全屏覆盖。
- 添加资料可用弹窗或独立资料页，但不得挤压会话主区。

移动端或窄屏：

- 默认只显示会话主区。
- 工作区侧栏通过菜单抽屉打开。
- 依据抽屉从右侧或底部打开。
- 输入框按钮减少为图标，长文字进入菜单。

## 10. 交互动效规则

动效只用于说明界面状态变化，不用于装饰。第一版需要设计，但必须克制：

- 侧边栏展开和完全隐藏：160-220ms，横向滑动，隐藏后不保留品牌窄栏或空白占位。
- 依据抽屉打开和关闭：160-220ms，从右侧进入；窄屏可从右侧或底部进入。
- 添加资料弹窗打开：180-240ms，轻微淡入和缩放；关闭时不需要复杂动画。
- 资料导入步骤切换：120-180ms，内容轻微淡入，不做页面大幅移动。
- 设置页打开：180-240ms，全屏淡入或轻微上移进入，返回时直接回到原会话。
- 聊天输入框左下角 `+` 菜单、自定义工具菜单：100-150ms，轻微淡入。
- 加载状态只允许使用文字、静态提示或非常轻的呼吸效果；没有真实后端进度时，不显示进度条。

动效边界：

- 必须支持系统“减少动态效果”设置，命中时动画降到最短或关闭。
- 不使用循环背景动画、彩色流光、弹跳、旋转等装饰性效果。
- 不通过动画暗示后端尚未支持的能力，例如实时导入进度、批次回滚、跨工作区同步。
- 动画不得改变主要布局尺寸，避免会话区、输入框和资料面板跳动。

## 11. 视觉规则

视觉方向为黑白灰轻量界面：

- 背景：白色或浅灰。
- 文字：黑色和深灰。
- 选中态：浅灰背景。
- 主按钮：黑底白字。
- 边框：浅灰。
- 错误：少量红色。
- 警告：少量黄色。

禁止：

- 大面积彩色渐变。
- 管理后台式数据卡片堆叠。
- 首页铺满说明文案。
- 用专业术语做按钮名。
- 用不可用能力占据主流程。

图标应优先使用通俗符号：菜单、加号、文件夹、文件、链接、齿轮、发送、返回、搜索。

## 12. 验收标准

正式实现进入验收时，应满足：

1. 应用默认进入会话首页。
2. 首页只显示当前工作区、会话、聊天输入和依据入口。
3. 普通界面使用“会话”，不使用“线程”。
4. 添加资料入口清楚可见。
5. 资料导入按五步流程运行。
6. 导入结果区分已加入、已更新、已跳过和失败。
7. 已导入资料可以浏览和预览。
8. 首页不展示检索调试、导入批次、工具历史和模型详细参数。
9. 回答依据默认收起，用户需要时打开。
10. 设置是全屏页，技术字段默认收起。
11. 练习从聊天 `+` 进入，不作为一级页面。
12. 不伪装后端未支持的跨工作区资料、文件级问答范围、实时导入进度和批次回滚。
13. 侧边栏、资料弹窗、依据抽屉、设置页和菜单具备克制的状态切换动效，并支持减少动态效果。

## 13. 后续实施建议

建议按以下顺序进入后续任务：

1. HTML 可交互预览：先验证首页、添加资料、依据抽屉和设置页手感。
2. 动效预览确认：验证侧边栏完全隐藏、弹窗、依据抽屉、设置页和菜单的切换是否足够轻。
3. 前端外壳改造：建立工作区侧栏、会话主区、依据抽屉。
4. 资料导入改造：按五步流程重组现有导入能力。
5. 设置页改造：把模型、保存、备份和外观收进全屏设置。
6. 练习工具化：把评估入口收进聊天 `+`。
7. 后端扩展任务：跨工作区资料、实时导入进度、会话置顶归档等另列 BACKLOG。

本文件确认的是目标体验和边界，不直接定义数据库迁移或 API 破坏性变更。
