# 用例定义

> 状态：Active
> Owner：RAG 团队
> Last Updated：2026-05-16

## 1. 主要用户场景

### UC-01 项目空间创建与导入

- 用户创建项目空间并选择本地目录。
- 系统扫描目录中的支持文件并建立文档与 chunk。
- 用户在无文件变更时重复扫描不应重复重建全部索引。

### UC-02 变更文件增量更新

- 用户新增/修改/删除文件后，系统在增量模式下只重建受影响内容。
- 删除文件应同步清理检索索引与来源记录，避免展示过期来源。

### UC-03 文档安全展示

- 用户打开已导入 Markdown 时，系统展示受控 HTML（禁止脚本和事件属性执行）。
- 文档仍保留原始内容、标准化内容、纯文本索引字段。

### UC-04 问答检索

- 用户输入问题后，系统返回含可读答案和来源列表。
- 若上下文不足，应明确提示“资料不足”，不虚构答案。

### UC-05 掌握度推进

- 用户基于知识点完成证据记录后，可推进 `MasteryRecord` 状态。
- 仅状态流转变更；最终可视化页面为后续工作。

## 2. 需求约束

- 优先使用本地流程；网络服务仅限模型调用。
- 所有用户可见文案优先“项目空间”。
- 失败场景应可继续执行其他文件处理（单文件失败不阻断全量导入）。
