## 1. Web MVP 主界面结构

当前默认入口是本地 Web MVP。前端采用左侧导航 + 独立视图的工作台布局，重点突出“项目问答”的主流程；资料库、掌握评估和设置不再堆叠在首页，而是通过左侧导航切换到各自视图。

```text
+----------------------+-----------------------------------------------+
| 左栏：主导航           | 当前视图                                      |
| - 品牌                 | 工作台：提问 / 回答 / 来源摘要 / 检索         |
| - 工作台               | 资料库：项目空间 / 导入 / 文件列表 / 预览     |
| - 资料库               | 评估：生成检查题 / 提交回答 / 查看反馈        |
| - 评估                 | 设置：新建项目空间                            |
| - 设置                 |                                               |
+----------------------+-----------------------------------------------+
```

### 1.1 信息密度原则

- 左栏只保留品牌和主导航，避免把项目管理功能混在首页。
- 工作台视图只承载首次引导、提问、回答、来源摘要和检索。
- 资料库视图承载项目选择、导入/同步、最近文件、文件预览和导入状态。
- 掌握评估与设置拆成独立视图，避免默认页面信息密度过高。
- 掌握评估视图在提交回答后显示 A+B 图表概览：左侧原生 SVG 雷达图展示派生能力画像，右侧得分环和标签展示本次命中/待补充要点。
- 点击左侧导航是视图切换，不是页面内锚点跳转或下拉展开。

```text
+---------------------------+
| 左侧导航（项目空间/知识库/问答/生成/设置）|

| 2) 项目列表或导入页面                |
| 3) 文件列表与进度                    |
| 4) 问答输入与来源显示                |
| 5) 生成表单与结果                    |
+---------------------------+
| 底部任务状态栏：进度、警告、错误码       |
+---------------------------+
```

## 3. 关键流程

- **导入前**：无项目空间时提示创建并阻断问答按钮。
- **导入中**：显示 scan/process/index 进度分段。
- **问答中**：答案和来源列表同步刷新，支持错误重试。
- **首次启动**：首次引导页显示安装依赖和路径配置。

## 4. 文案统一

- 面向用户文案统一使用“项目空间”。
- 后台兼容字段不进入用户提示文案。
