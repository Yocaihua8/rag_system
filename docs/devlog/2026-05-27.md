## 1. Done（今日完成）

- B-141K 第一片：扩展 `frontend/src/api/imports.js`、`DocumentImportPanel` 和 `App.vue`，资料库视图支持当前项目目录导入预检，复用既有 `GET /api/import/preview`；预检只展示可导入数、跳过数和跳过原因，不创建导入批次、不刷新文档列表。
- B-141L 第一片：新增 `frontend/src/api/document-collections.js` 和 `DocumentCollectionPanel`，资料库视图支持文档集合只读筛选，复用既有 `GET /api/document-collections` 与 `GET /api/documents?collection_id=...`；当前不提供集合新建、删除、加入或移出文档入口。
- B-141M 第一片：扩展 Vue 文档集合 helper、`DocumentCollectionPanel` 和 `App.vue`，资料库视图支持文档集合新建和删除，复用既有 `POST /api/document-collections` 与 `POST /api/document-collections/delete`；删除集合前提示“集合内文档不会被删除”，删除当前筛选集合后回到全部文档。
- B-141N 第一片：扩展 Vue 文档集合 helper、`DocumentCollectionPanel` 和 `App.vue`，资料库视图支持文档集合重命名，复用既有 `POST /api/document-collections/update`；重命名成功后刷新集合列表。
- B-141O 第一片：扩展 Vue 文档集合 helper、`DocumentListPanel` 和 `App.vue`，资料库文档列表支持单文档加入集合和从当前集合移出，复用既有 `POST /api/document-collections/items/add` 与 `POST /api/document-collections/items/remove`；成功后刷新集合列表和当前文档列表。

## 3. Decisions

- B-141K 只迁移 Vue 侧预检入口，不改后端 API 和数据库 schema；原因是后端只读预检契约已存在，本片目标是补齐 Vue 资料库导入区域。
- B-141L 先迁移集合列表与筛选，不迁移集合 CRUD 和文档加入/移出；原因是筛选复用现有 `collection_id` 文档列表契约，风险低且能先补齐资料库浏览体验。
- B-141M 只迁移集合新建/删除，不迁移集合重命名、加入文档或移出文档；原因是新建/删除可独立复用既有接口，加入/移出需要改文档列表项操作区，适合后续单独切片。
- B-141N 只迁移集合重命名，不迁移文档加入/移出集合；原因是重命名只影响集合元数据和集合列表刷新，加入/移出需要改文档列表项操作区，适合后续单独切片。
- B-141O 只迁移单文档加入/移出集合，不迁移批量选择、拖拽、删除文档或问答按集合过滤；原因是 items add/remove 已有后端契约，本片可限定在文档列表项操作区和刷新流。
- 技术栈迁移整体完成前不推送；本片按计划继续本地提交。
