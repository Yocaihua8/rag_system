# UI 页面结构说明

> 状态：Active
> Owner：RAG 团队
> Last Updated：2026-06-04
> Scope：Knowledge Island Web MVP 前端页面布局与核心交互
> Related：docs/requirements/functional-modules.md, docs/design/api-spec.md

## 1. 页面清单

| 页面 | 目标用户 | 主要目标 | 优先级 |
|------|----------|----------|--------|
| 工作台（主页）| 所有用户 | RAG 问答、多轮对话、Agent 工具、检索复盘 | P0 |
| 资料库 | 所有用户 | 导入文档、管理集合、查看健康状态、批次历史 | P0 |
| 评估 | 学习者 | 生成题目、逐题作答、查看掌握情况、答题记录和待复测列表 | P1 |
| 设置 | 技术用户 | 配置 LLM、模型 Profile、Prompt 预设、数据备份 | P1 |

## 2. 布局表达约定

```text
┌─────────────────────────────┐
│  [区域名称]  表示 UI 功能区  │  ← 方括号为区域标签，非代码
└─────────────────────────────┘
```

导航栏固定在顶部，页面通过顶部标签切换视图（SPA，不刷新页面）。B-149 后顶部采用海图志 `masthead`：左侧为 IslandMark 与 `知识岛 · Knowledge Island`，副标题显示 `海图志 · Vol. I · 2026 · 当前项目名`；右侧为工作台 / 资料库 / 评估 / 设置四个导航按钮和 `FOL. I-IV` 页码。

页面支持系统深色偏好；用户也可以通过侧栏主题按钮在浅色 / 深色之间手动切换，手动选择会保存到浏览器 `localStorage`。

## 3. 工作台（主页）

**页面目标**：支持多轮对话式 RAG 问答，查看来源和工具结果。

**B-149 当前结构**：工作台采用海图志三列 `.page` 布局：左列是 `.session-item` 会话列表，中列是 `.conv-col` 对话流和 `.composer` 输入区，右列是 Agent 工具与 `.debug` 检索调试。回答区使用 `.turn.assistant`、`.obs`、`.sources` 和 `.feedback` 展示回答正文、可观察性元数据、来源和反馈按钮。

```text
┌────────────────────────────────────────────────────────────────┐
│  [顶部导航栏]  工作台 · 资料库 · 评估 · 设置  |  [项目选择器] ▼│
├──────────────┬─────────────────────────────────────────────────┤
│ [会话列表]   │  [对话区域]                                     │
│ + 新建会话   │   ┌──────────────────────────────────────────┐  │
│ ─────────── │   │ [用户消息]  消息内容                      │  │
│ ● 会话 1    │   │ [AI 回答]  回答文本（含 Markdown）        │  │
│   会话 2    │   │ [可观察性条] hits:3 top_k:5 model:api     │  │
│   会话 3    │   │ [来源列表]  ① 文件A · ② 文件B            │  │
│             │   │ [反馈按钮]  👍 有用  👎 无用  ⚠ 来源不准  │  │
│             │   └──────────────────────────────────────────┘  │
│             │   [工具建议横条]  ⚡ 建议运行 search_sources     │
│             │   [输入框]  在此输入问题...        [发送 ▶]      │
├──────────────┴─────────────────────────────────────────────────┤
│  [Agent 工具面板]  ▼ 展开                                       │
│    project_overview [运行]  |  search_sources [查询词] [运行]   │
│    最近运行历史：search_sources · 成功 · "RAG 检索流程"         │
├────────────────────────────────────────────────────────────────┤
│  [检索调试区]  ▼ 展开                                           │
│    top_k: [5]  min_score: [0.3]  [关键词 ✓] [向量 ✓]  [诊断] │
│    [保存为检索复盘]                                             │
└────────────────────────────────────────────────────────────────┘
```

**关键交互**：
- 切换会话后重新加载该会话聊天记录（`GET /api/chat/messages?session_id=`）
- 发送问题后按钮进入 loading 状态，禁用重复提交，并显示“取消”按钮关闭当前 EventSource 流
- 当模型设置或默认 Profile 使用云端 API 且 Key 已配置时，输入区在发送前提示“本轮会把检索到的来源片段发送给云端模型”；本地模式或未配置 Key 时不显示该提示
- 来源不足时显示建议工具横条，用户点击才触发工具运行（不自动执行）
- 工具运行成功后，可一键引用 `tool_run_id` 到下一轮问答
- 回答通过 `/api/answer/stream` 的 SSE `token` 事件逐步渲染，完成后用 `done` 事件刷新来源、观察性和聊天记录

**空状态**：暂无会话时显示"点击 + 新建会话开始问答"引导文案

**错误状态**：API 不可用时显示可恢复提示，说明下一步操作（检查服务 / 刷新）

## 4. 资料库

**页面目标**：让用户管理知识来源，查看知识库健康状态。

**B-149 当前结构**：资料库顶部使用 `.dashboard` 展示文档、集合、导入批次等轻量指标；中部使用 `.import-row` 并排承载导入操作和导入批次历史；集合筛选使用 `.coll-tabs`；文档列表使用 `.doc-table`，并保留预览、加入集合、移出集合和删除文档入口。

```text
┌────────────────────────────────────────────────────────────────┐
│  [顶部导航栏]                                                   │
├────────────────────────────────────────────────────────────────┤
│  [项目健康仪表盘]                                               │
│   文档 42  Chunk 386  向量 386  聊天 127  工具运行 8  复盘 3   │
│   最近活动：2026-05-25 14:32                                    │
│  [检索健康指示]  ✅ 向量已生成  /  ⚠ 仅关键词  /  ❌ 无 Chunk │
├────────────────────────────────────────────────────────────────┤
│  [导入操作区]                                                   │
│   [同步目录]  [上传文件]  [文本笔记]  [URL 摘录]  [剪贴板]     │
│  [导入预检]  预计新增 12 / 跳过 3（格式不支持 2 / 过大 1）     │
│  [最近导入批次]  目录同步 · 完成 · 新增12/跳过3 · 05-25 14:31 │
├────────────────────────────────────────────────────────────────┤
│  [集合筛选]  全部(42) | 未分组(15) | 后端项目(18) | 笔记(9)   │
│             [+ 新建集合]                                        │
│  [文档列表]                                                     │
│   📄 README.md          note:        2026-05-25   [删除]       │
│   📄 api-spec.md        local:docs/  2026-05-24   [删除]       │
│   📄 架构笔记            note:        2026-05-23   [删除]       │
└────────────────────────────────────────────────────────────────┘
```

**关键交互**：
- 点击"同步目录"后按钮置为 loading，返回导入统计（新增/更新/跳过/失败）
- 当前项目区域显示后端返回的绑定目录 `root_path`；目录不存在时显示“项目目录不存在”
- 导入前可点击"预检"查看估算结果，不写数据库
- 集合筛选即时过滤文档列表，无需刷新
- 删除文档前二次确认

**空状态**：无文档时显示"从上方选择导入方式开始建立知识库"

## 5. 评估页

**页面目标**：出题、作答、查看掌握等级和错题记录。

**B-149 当前结构**：评估页使用 `.eval-frame` 聚合题目、统计摘要、`.question`、`.answer-box`、提交动作和 `.verdict` 结果卡；答题记录与待复测列表保留在同一 frame 下方。

```text
┌────────────────────────────────────────────────────────────────┐
│  [顶部导航栏]                                                   │
├────────────────────────────────────────────────────────────────┤
│  [评估入口]                                                     │
│   当前项目：知识岛 Web MVP     [开始新一轮评估]                 │
│   上次评估：2026-05-24  已掌握 3 / 基本理解 2 / 需补充 1       │
├────────────────────────────────────────────────────────────────┤
│  [题目区]                                                       │
│   第 2 / 5 题                                                   │
│   ─────────────────────────────────────────────────────────── │
│   请描述 RAG 混合检索的流程，以及关键词检索和向量检索如何合并。 │
│   ─────────────────────────────────────────────────────────── │
│   [答案输入框（多行）]                                          │
│                                                                 │
│   [跳过]                              [提交回答]               │
├────────────────────────────────────────────────────────────────┤
│  [评估结果]（提交后显示）                                       │
│   🟡 基本理解    参考来源：docs/design/search.md § 3           │
│   补充说明：回答提到了关键词检索但未说明得分合并策略           │
│   [继续追问]              [下一题 →]                           │
├────────────────────────────────────────────────────────────────┤
│  [错题记录]  ▼ 展开                                             │
│   • 向量化降级策略（暂未掌握）— 下轮优先复测                   │
└────────────────────────────────────────────────────────────────┘
```

**当前实现**：Web 前端已支持一轮多题评估的进度展示、逐题提交、单题反馈、答题记录和待复测列表；跨刷新持久化的错题复测入口仍待后续任务细化。

**空状态**：项目无文档时提示"请先在资料库导入文件再开始评估"

## 6. 设置页

**页面目标**：配置 LLM 连接、模型 Profile 和 Prompt 预设。

**B-149 当前结构**：设置页使用 `.settings-frame` 包裹模型设置、Model Profile 和 Prompt 预设三个 `.section`；基础模型设置使用 `.field-grid` 呈现 API 地址、模型名称、API Key 状态和测试 / 保存动作。

```text
┌────────────────────────────────────────────────────────────────┐
│  [顶部导航栏]                                                   │
├────────────────────────────────────────────────────────────────┤
│  [模型设置]                                                     │
│   API 地址  [https://api.deepseek.com/v1        ]              │
│   模型名称  [deepseek-chat                      ]              │
│   API Key   [已配置 env:DEEPSEEK_API_KEY        ]              │
│   [测试连接]                    [保存设置]                      │
│   ✅ 连接成功 · deepseek-chat · api                            │
├────────────────────────────────────────────────────────────────┤
│  [模型 Profile]                                                 │
│   ● DeepSeek（默认）  deepseek-chat  api.deepseek.com   [编辑] [删除] │
│     OpenAI GPT-4o    gpt-4o         api.openai.com      [编辑] [删除] │
│   [+ 新增 Profile]                                             │
├────────────────────────────────────────────────────────────────┤
│  [Prompt 预设]（当前项目：知识岛 Web MVP）                      │
│   ● 项目问答（默认）   简洁技术回答风格                [编辑] [删除] │
│     代码解释           逐步解释代码意图               [编辑] [删除] │
│     学习复盘           苏格拉底式追问                 [编辑] [删除] │
│   [+ 新增预设]                                                 │
├────────────────────────────────────────────────────────────────┤
│  [数据备份]                                                     │
│   [导出当前项目备份]           [从备份文件恢复]                 │
└────────────────────────────────────────────────────────────────┘
```

**关键交互**：
- 测试连接不改变配置，仅验证可达性
- API Key 回显只显示来源引用（`env:DEEPSEEK_API_KEY`），不显示明文
- Profile 设为默认后，下次问答优先使用该 Profile

## 7. 空状态与错误状态约定

| 场景 | 展示文案 | 下一步引导 |
|------|----------|------------|
| 未创建项目 | "尚无项目空间" | 跳转设置页创建 |
| 项目无文档 | "知识库为空" | 跳转资料库导入 |
| 检索无命中 | "未找到相关来源" | 建议换关键词或补充资料 |
| API 不可用 | "本地服务暂时不可访问" | 提示刷新或检查 `backend/app.py` |
| LLM 未配置 | "未配置模型，使用本地片段回答" | 跳转设置页配置 API |
