# 系统设计总览

> 状态：Active
> Owner：RAG 团队
> Last Updated：2026-05-16
> Scope：知识岛桌面客户端与知识流水线
> Related：`design/architecture-overview.md`

## 1. 设计结论

知识岛采用以桌面客户端为边界的“本地优先 AI 助手”系统设计。核心目标是降低“文件到知识”的认知成本，且不要求联网数据库即可完成常态使用。

```text
[本地文件系统] -> [文档入库/标准化]
                   -> [持久化数据库 + 向量库]
                   -> [检索 + 生成]
                   -> [桌面 UI 展示/交互]
```

## 2. 系统边界

- 输入边界：本地目录与本地设置。
- 处理边界：本地 SQLite、ChromaVector/Numpy Store、Ollama 或 OpenAI 兼容模型。
- 输出边界：桌面 UI 回显、索引结果、问答与来源信息。

## 3. 关键协作点

- 文档标准化、检索、问答是同一个业务链路：`ingest -> split -> embed -> index -> retrieve -> answer`。
- 项目空间与知识模型是长期演进入口；现有兼容层 `Workspace` 保持稳定。

## 4. 非功能性约束

- 大文件扫描不应阻塞主线程。
- 单文件解析失败不应中断整批扫描；应记录并继续。
- HTML 渲染必须安全边界化，禁止脚本执行。
