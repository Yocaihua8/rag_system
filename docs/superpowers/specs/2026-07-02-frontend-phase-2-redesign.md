# 第二阶段前端重设计规格

> 状态：Draft（已对齐 HTML 交互预览）
> Owner：RAG 团队
> Last Updated：2026-07-02
> Scope：Knowledge Island 第二阶段前端信息架构、页面结构、交互边界与验收标准
> Related：docs/previews/phase2-frontend-preview/index.html, docs/design/ui-wireframes.md, docs/features/frontend-engineering.md, docs/design/api-spec.md, docs/requirements/functional-modules.md, docs/BACKLOG.md

## 1. 目的

第二阶段重设计以 Tauri 桌面体验优先，浏览器 Web 版继续作为开发和回归入口。当前前端承接了 B-141/B-142 迁移期结构，页面中仍有大量工程迁移文案、专业术语和并列面板。第二阶段的目标是把 Knowledge Island 调整为更轻量的本地资料聊天工具：用户先管理资料，再围绕资料进行聊天、整理和理解检查。

本设计只定义第二阶段目标和边界，不表示相关功能已经在正式 Vue 前端中实现。当前可交互预览位于 `docs/previews/phase2-frontend-preview/index.html`，正式实现时以该预览的交互结构为第一版蓝图。

## 2. 设计结论

默认首页是 `聊`。它参考 Codex 的侧边栏和线程管理，也参考 AnythingLLM 的资料聊天模式，但不照搬专业术语。首页主区域保持极简，用户只看到当前资料状态、聊天输入框、`加入资料`、`更多`、回答引擎、思考深度和 `发送`。

一级入口保留三类：

| 入口 | 定位 | 第一版范围 |
|------|------|------------|
| `聊` | 默认首页，围绕当前资料聊天 | 工作区、线程、聊天、回答依据、聊天工具 |
| `库` | 资料管理弹窗 | 加入库、选择资料、连接工作区、导入结果 |
| `设` | 全屏设置页 | 回答引擎、在线服务、资料保存位置、备份恢复、外观 |

独立一级 `评估/练习` 不再保留。已有评估能力收缩为聊天工具，例如 `+ -> 练习与小测`，或用户直接输入“根据当前资料出几道小题”。

视觉方案采用黑白灰中性配色。主界面不再使用绿色或蓝色作为品牌强调色；选中态使用浅灰底和黑色文字，主按钮使用黑底白字，错误和警告状态保留必要提示色。

## 3. 命名边界

当前 `docs/style-guide.md` 写明项目使用“项目空间”作为对外文案。第二阶段设计提出在新 UI 中使用“工作区”或“资料空间”表达同一类用户概念，内部 API 和数据库仍保留现有 `project_id` / project 命名。

命名调整属于规划中的 UI 文案变更。实施时必须同步更新相关文档，否则不能把“工作区”表述写入当前已落地行为文档。

术语替换原则：

| 当前术语 | 第二阶段普通界面文案 | 说明 |
|----------|----------------------|------|
| Agent 工具 | 更多工具 | 不暴露 Agent 概念 |
| 检索调试 | 检查答案依据 | 面向普通用户说明用途 |
| Prompt 预设 | 回答方式 | 放入设置或高级区域 |
| 模型 Profile | 回答配置 | 不在首屏出现 |
| 模型 / LLM | 回答引擎 | 首屏和设置主路径使用普通文案 |
| API 接入 | 在线回答 | 技术字段仅保留在高级详情或表单内部 |
| Chunk / 向量 / top_k / RAG | 不默认展示 | 仅高级检查可见 |

全局术语替换不纳入第一版固定范围。第一版只要求首屏和核心流程不出现专业词，其他历史文案随具体切片按需同步。

## 4. 信息架构

### 4.1 默认首页：聊

默认首页包含：

- 左侧侧边栏：当前工作区、线程列表、新建线程、线程切换、加入资料入口
- 中间主区域：当前聊天、输入框、`+` 工具、回答引擎、思考深度、`发送`
- 右侧依据栏：默认收起，回答后可展开，也保留边缘把手

左侧侧边栏参考 Codex 的聊天记录组织方式。用户可以在同一工作区下创建多个线程，切换后恢复对应聊天历史。工作区列表默认不平铺，点击当前工作区后以弹层切换，减少首页信息密度。

输入框左下角 `+` 内包含：

- 上传临时文件
- 加入资料
- 练习与小测
- 整理要点

输入框左下角 `+` 旁边保留快捷工具图标，不需要先点 `+` 才能打开。

快捷工具内再收起：

- 快捷指令
- 常用指令
- 连接工具
- 对比回答

这些能力复用现有后端能力或后续接口，不在首屏平铺。

移动端默认显示聊天主区和顶部栏，不直接铺开完整侧边栏。顶部栏保留侧边栏按钮，点开后以抽屉显示当前工作区、线程列表和加入资料入口。

### 4.2 资料管理弹窗：库

`库` 不承担问答主流程，只负责资料来源和资料管理。第一版采用弹窗，而不是二级侧边栏，避免挤压聊天区域。弹窗分为两步：

- `加入库`：左侧是资料来源，右侧是来源表单和本次加入结果
- `选择资料`：主左侧侧边栏切换为工作区选择模式，弹窗右侧显示资料夹筛选和资料列表

用户流程必须是“先加入库，再从库中选择资料加入工作区”。不能在上传页直接要求用户选择工作区。
每次打开资料弹窗时，`选择资料` 默认选中当前侧边栏里的工作区；进入 `选择资料` 后，主左侧侧边栏从线程列表切换为工作区列表，用户在这里选择资料要加入的工作区。
`加入库结果` 默认折叠，只在用户展开或刚完成加入库后显示，避免弹窗像管理后台。

第一版只展示普通用户能理解且同阶段可用的来源：

- 本机文件夹：一次导入整个文件夹
- 文件上传：上传单个或多个文件
- 文本笔记
- 网页摘录 / 单页抓取
- 更多来源：收起代码仓库、笔记库、压缩包等稍后支持能力，不作为可用主入口

GitHub、GitLab、YouTube、Confluence、Obsidian、Notion、Drupal Wiki、Paperless-ngx 等连接器如果后端未实现，不能在第一版作为可用入口展示。可收进“更多来源”的稍后支持说明，或暂不显示。

### 4.3 设置页：设

`设` 第一版是全屏覆盖页，带返回按钮。左侧是设置菜单，右侧一次只显示一个设置页面，不做长滚动页。第一版提供主要配置能力：

- 回答引擎：本机回答、本机快速、在线回答
- 在线服务：DeepSeek / OpenAI-compatible 地址、服务名、访问密钥状态、连接测试
- 资料保存位置：显示当前保存位置；修改能力如涉及迁移需另拆任务
- 备份导出 / 恢复
- 深色模式
- 回答方式和高级检查入口

访问密钥不回显明文，只展示来源和状态。
服务地址、在线服务名等技术字段默认收进 `连接详情`，不在设置主页面直接展开。

### 4.4 右侧依据栏

右侧 `依据` 栏默认收起。收起时保留边缘把手或顶部按钮，保证用户能发现。展开后按场景展示：

- 回答参考了哪些资料
- 引用片段
- 导入失败或跳过原因
- 检查答案依据
- 工具运行详情

普通用户默认只看到“这个回答参考了哪些资料”。高级参数只在高级检查里出现。抽屉应支持按钮打开、回答卡片打开、边缘把手打开，以及 `Esc` 收起。

## 5. 核心用户流程

### 5.1 首次使用

用户打开桌面应用后进入 `聊`。如果没有工作区，侧边栏显示“新建工作区”；主区域提示“加入资料”。用户可以不配置模型，系统仍可用本地资料片段回答。

### 5.2 工作区与线程

侧边栏顶部显示当前工作区。用户可创建、切换、重命名和删除工作区。线程列表位于工作区下方，支持新建线程、切换线程、重命名和删除。

线程归档不作为第一版必做项。如果后端没有归档字段，不能把删除伪装成归档。

### 5.3 聊天问答

用户在输入框提问。回答流式显示在主区域，回答完成后可展开右侧 `依据` 查看来源。回答反馈保留，但不抢占首屏。

### 5.4 聊天工具

用户通过输入框左下角 `+` 触发临时文件、加入资料、练习与小测、整理要点。`+` 旁边的快捷工具图标打开快捷指令、常用指令、连接工具和回答对比。`练习与小测` 使用现有评估能力或后续聊天工具封装，不再作为独立一级页面。

### 5.5 资料导入

用户点击 `加入资料` 或 `库` 后进入资料弹窗。弹窗先进入 `加入库`，用户可选择本机文件夹、文件、笔记或网页摘录。加入完成后进入 `选择资料`，主左侧侧边栏切到工作区列表，弹窗右侧显示资料夹和资料列表。当前后端仍以 `project_id` 为资料边界，第一版不伪造“已连接到多个工作区”或“连接完成”状态；真实跨工作区连接拆到 B-157。

导入结果使用普通文案：

- 已加入
- 已更新
- 未加入
- 失败原因

### 5.6 设置回答引擎

用户在 `设` 中选择本机回答、本机快速或在线回答。回答引擎不可用时，界面提示仍可管理本地资料，并提供去设置连接回答引擎的入口。

## 6. 页面与组件边界

### 6.1 应用外壳

`AppShell.vue` 应从简单导航壳改为桌面工作区壳：

- 左侧工作区 / 线程侧边栏，可折叠
- 中间主内容
- 右侧依据抽屉，可折叠并带把手

`App.vue` 当前承担大量状态中转。实施时应该拆出工作区、线程、聊天、资料库和设置相关 composable 或 store，避免继续扩大单文件。

### 6.2 聊页面

`WorkbenchView.vue` 改为默认聊天首页，保留：

- 聊天消息流
- 输入框
- 流式回答与取消
- 回答反馈
- `加入资料`
- `更多`
- 右侧依据栏触发

检索调试、复盘、工具历史、模型对比不再平铺在首屏。

### 6.3 线程侧边栏

`ChatSessionPanel.vue` 扩展为线程管理侧边栏：

- 新建线程
- 线程列表
- 搜索线程
- 重命名 / 删除
- 当前工作区显示
- 工作区切换弹层

搜索可先前端过滤当前列表；大量历史搜索再补后端接口。

### 6.4 库弹窗

第一版优先新增或重组为 `LibraryModal` 这类弹窗组件；来源选择和工作区资料选择可以先内联在弹窗和侧栏中，后续复杂后再拆成独立组件。`库` 导航点击应打开弹窗，不切换到挤压聊天的二级侧边栏。现有 `LibraryView.vue` 可作为后续完整资料浏览页的承接处，但不承担首页问答主流程。

当前第一版实际组件边界：

- `LibraryModal`：资料来源、资料夹筛选和资料列表先放在同一个弹窗组件内
- `WorkspaceSidebar`：在选择资料时切换到工作区选择模式
- `ImportResultList`：本次加入库结果，默认折叠

### 6.5 设页面

`SettingsView.vue` 重排为全屏覆盖设置：

- 回答
- 资料
- 外观

每个设置项对应一个页面，左侧点击切换，右侧只显示当前页。不要做所有设置堆叠的长滚动页。

设置页必须保留访问密钥安全规则。

## 7. 接口与数据边界

### 7.1 优先复用现有后端

第一版优先复用现有 API：

- 项目空间
- 聊天会话
- 流式问答
- 文档导入
- 导入批次
- 文档列表
- 回答引擎设置
- 回答配置
- Prompt 预设
- 评估
- 工具运行记录

### 7.2 可能需要另拆任务的接口

以下能力如现有接口不足，需要单独设计：

- 线程归档
- 跨大量历史的线程搜索
- 后端提供连接器元数据
- 修改资料保存位置和迁移既有资料
- 管理多个本机回答来源
- `练习与小测` 作为聊天工具消息展示

### 7.3 数据库边界

第一版不主动修改数据库。以下需求会触发 Schema 风险，必须单独确认：

- chat session 增加 `archived_at`
- 工具调用结果和聊天消息统一建模
- 保存来源配置
- 修改资料根目录并迁移已有资料
- 评估结果绑定聊天线程

### 7.4 安全边界

`更多工具` 中的能力继续遵守只读边界。禁止开放 shell、任意命令、文件写入或自动访问外部服务。更多来源中的代码仓库、笔记库、压缩包等能力在后端未实现前只能显示为稍后支持，不能保存 token、用户名、密码或 SSH Key。

## 8. 验收标准

第一版完成时必须满足：

1. 打开应用默认进入 `聊`。
2. 左侧有工作区和线程管理，支持新建和切换线程。
3. 主区域是简洁聊天输入，不展示迁移期说明。
4. 用户能提问、接收流式回答、取消回答、查看回答来源。
5. `加入资料` 能打开资料弹窗，并先加入库。
6. `库` 是资料管理弹窗，不承担问答主流程，也不挤压聊天区域。
7. `选择资料` 时主左侧侧边栏必须切换为工作区列表，弹窗右侧必须是资料夹筛选和资料列表；后端未实现跨工作区连接前，不显示伪造的连接成功状态。
8. `库` 只展示真实可用或同阶段实现的资料来源；计划内来源收进“更多来源”并不可直接使用。
9. `设` 是全屏覆盖页，带返回按钮，左侧菜单切换右侧单页内容。
10. `设` 能完成本机回答状态查看、在线回答连接测试、备份恢复和深色模式等主要配置。
11. 独立一级 `评估/练习` 入口取消，能力收进聊天工具。
12. 首屏不出现 RAG、Agent、向量、Chunk、top_k、Prompt、Profile、API Key 等专业词。
13. 回答引擎在普通界面展示为“本机回答 / 本机快速 / 在线回答”，技术模型名只放在设置详情里。
14. 右侧依据是抽屉，默认收起，保留可发现的把手或按钮。
15. 移动端默认是聊天主区和顶部栏；侧边栏以抽屉打开，不能把聊天压到页面下方。
16. 不引入假数据，不把未实现来源伪装成可用。
17. Tauri 桌面体验优先，浏览器版保持可用和可测试。

## 9. 实施切片建议

建议分为以下切片：

1. 设计和文档切片：同步第二阶段设计到项目正式文档，并锁定 BACKLOG 条目。
2. 外壳和导航切片：重做 `AppShell.vue`，建立左侧线程侧边栏、中间内容、右侧依据抽屉。
3. 聊天首页切片：重组 `WorkbenchView.vue`，把调试、复盘、工具、回答对比收进 `+` 或右侧抽屉。
4. 资料库弹窗切片：新增或重组资料弹窗组件，实现加入库、选择资料和目标工作区选择；真实跨工作区连接另列后端任务。
5. 设置页切片：重排本机回答、在线回答、备份恢复、深色模式。
6. 术语与测试同步：每个切片内按需更新，不作为独立全局替换切片。

以下内容不纳入第一版固定范围，后续由实施计划逐项决定：

- 是否全局替换所有用户可见专业术语
- 是否批量重写现有前端静态断言测试
- 是否统一重命名组件，或仅调整展示文案
- 是否将 `评估` 从路由和状态中彻底移除，或只从一级导航隐藏

## 10. 正式 Vue 迁移对照

后续进入正式代码编写时，优先按当前文件职责迁移，不做静态假界面替换：

- `frontend/src/components/AppShell.vue`：承接三栏外壳、一级入口、侧边栏收起、设置全屏覆盖。
- `frontend/src/views/WorkbenchView.vue`：承接 `聊` 首页，保留聊天真实数据流，把调试、工具、回答对比收进 `+` 或右侧依据抽屉。
- `frontend/src/components/QuestionComposer.vue`：扩展输入框左下角 `+`、自定义工具、回答引擎和思考深度入口。
- `frontend/src/views/LibraryView.vue` 与资料组件：拆出 `LibraryModal`、来源选择、加入库结果、工作区资料选择，不再作为问答主流程页面。
- `frontend/src/views/SettingsView.vue`：拆成全屏设置壳和 `回答 / 资料 / 外观` 单页内容，保留访问密钥不回显规则。
- `frontend/src/components/AnswerPanel.vue` 与 `SearchDebugPanel.vue`：整理为依据抽屉内容，默认收起，按当前回答或当前消息展示依据。
- `frontend/src/state/app-state.js`：后续不能继续把 `库` 当成普通页面切换；`库` 应作为弹窗状态，`评估` 不应作为一级入口状态暴露。
- `frontend/src/App.vue`：弹窗、抽屉、设置全屏和聊天状态需要拆到清晰的 composable 或 store，避免把第二阶段状态继续堆在单文件。

若回答引擎或思考深度需要单次对话生效，必须先确认后端接口是否支持；不支持时只能连接到现有默认回答配置，不能在前端伪装成已生效。
资料导入也必须分清“加入库”和“连接到工作区”：旧导入组件里直接依赖 `projectId` 的逻辑，不能原样搬进 `加入库` 步骤。
依据抽屉的第一版数据来源默认为最新回答；若要点击历史消息查看依据，需要先确认消息接口是否稳定返回来源快照。

## 12. 设计记录

本设计由以下已确认决策收敛而来：

- 第二阶段优先级：首次使用闭环、日常效率、视觉与术语。
- 默认用户：普通用户能理解，高级能力隐藏到更多或高级设置。
- 运行形态：Tauri 桌面优先。
- 首屏：聊天工具方向，最小按钮。
- 视觉：采用黑白灰中性配色，不再沿用旧绿色或蓝色主色。
- 练习：不作为一级页面，交给聊天工具。
- 资料库：弹窗式资料管理，先加入库，再选择资料和目标工作区；真实跨工作区连接等待 B-157。
- 设置：第一版提供本机回答、在线回答、资料保存、备份恢复和外观。
