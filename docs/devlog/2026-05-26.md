## 3. Decisions

- 暂不新增 `rank_bm25` 运行时依赖 → 原因：项目规则要求 Web MVP 不新增必需运行时依赖，当前内置 BM25 公式已能覆盖 B-127 的排序问题
- `api.py` 拆分先做蓝图、后做代码迁移 → 原因：当前工作区已有大量未提交改动，先冻结目标结构和验收基线可降低后续批量移动路由时的回归风险
- `api_support.py` 只抽取跨路由真实复用 helper；回答链路单独抽到 `answer_api.py` → 原因：`POST /api/answer` 与 SSE 共享上下文准备逻辑，放在独立模块可避免 `routes/answers.py` 与 `api.py` 循环依赖
- B-139 先保留 `webapp.api.dispatch()` 兼容入口 → 原因：先替换 HTTP 运行时，避免同时重写 61 个 API 契约；后续如要原生 FastAPI Router，可另拆 BACKLOG
- B-140 不做登录页、不做用户表、不做 RBAC → 原因：当前仍是本地单用户阶段，先用可选认证保护远程访问边界，避免提前引入数据库迁移和权限模型
- B-141 采用分阶段迁移 → 原因：当前 legacy 前端功能较多，先建立 Vue/Vite 构建链和后端托管策略，再逐页迁移业务 UI，可降低回归面
